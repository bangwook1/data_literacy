# 1️⃣ 개념정리

## 01. 자꾸만 늘어가는 '방법맨'이란

```
✅ 학습 목표 :
* '방법맨'이란 무엇인지 설명할 수 있다.
* 문제 해결보다 기법 자체에 집착하게 되는 사고 방식의 위험성을 설명할 수 있다. 
```

- 문제정의에서 **원인과 해결방안이 섞여 있는 경우**가 많다.  
  → **문제 → 원인 → 해결방안** 순으로 접근해야 한다.  
- **무작정 해결방안에 달려들어서는 안 된다.**  
  참신한 아이디어로 곧장 해결하려 하기보다, **원인 분석이 우선**되어야 한다.  
- **해결방안은 문제 자체가 아닌 ‘원인’에 적용해야 한다.**  
- 예시 시간외 근무가 많다 → 근무를 줄이자! ❌<br>
→ 어떤 요일에 많은가? 와 같은 데이터 분석으로 접근해야 함 ✅

- 왜 다른 방법이 아닌 **그 방법이어야 하는지**,  
기대 가능한 결과나 효과적인 결과가 나올 **근거가 어디 있는지** 고민해야 한다.  
- **주관이 과도하게 개입되어 비약이 있다면**,  
단순히 방법만 찾는 **‘방법맨’**일 수 있다.

## 02. 방법맨이 돼버리는 구조

```
✅ 학습 목표 :
* 방법맨이 되어버리는 구조적 요인을 이해한다.
* 분석 기법보다 '문제 인식 -> 사고 구조 설계'가 우선임을 인식한다. 
```
- “생각한다”는 것이 **참신한 아이디어를 떠올리는 것인지**,  
아니면 **논리적인 전개를 구성하는 것인지** 구분해야 한다.  
- 생각하는 과정에서 **참신함만 강조하고 논리성을 잊기 쉽다.**  
- **참신함이 항상 좋은 것은 아니다.**  
→ 논리적 사고와 구조적 접근이 더 중요하다.

## 03. '왜(원인)' 그런지 끈질기게 생각하라

```
✅ 학습 목표 :
* 기법을 적용하기 전에 '무엇을, 왜 분석하는가'를 지속적으로 질문하는 태도를 기른다.
* 단순 결과에 안주하지 않고 원인을 집요하게 파고드는 분석 사고력을 기른다. 
```

- 생각하는 부분이 **문제인지, 원인인지, 해결방안인지**를 명확히 구분해야 한다.  
- **문제와 원인이 독립되어 있지 않은 경우도 많다.**  
- 원인 후보를 정한 뒤에도 **본질적인 원인까지 파고들어야 한다.**

**예시: 가게 매출 하락의 원인 분석**

1) 경쟁사 가격 인하

2) 방문객 수 감소 → 왜? 홈페이지 접속자 감소 → 왜? 관리 부실 → 왜? 직원 부족

3) 고객 대응 부실 → 왜? 매장 직원 수 부족

> - **표면적 요인 외에도 잠재적, 구조적 원인도 포함하면 더 좋습니다.**
>
> **3. 해결 방안 도출 : 위에서 제시한 각 원인에 대해, 현실적으로 적용 가능한 대응 방안을 1:1로 매칭하여 제시해보세요.**
